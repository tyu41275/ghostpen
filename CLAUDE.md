# ghostpen

Personal developer blog by Tim Yu. Built on Next.js 14 + Contentlayer2 + Tailwind CSS, deployed on Vercel. Originally forked from Tailwind Nextjs Starter Blog v2.

GitHub: tyu41275/ghostpen

## Architecture

```
app/                    # Next.js 14 app directory (pages, layouts, API routes)
  blog/                 # Blog listing and individual post pages
  about/                # About page
  tags/                 # Tag listing and per-tag post pages
  projects/             # Projects page
data/
  blog/                 # .mdx blog posts (YAML frontmatter + MDX body)
  authors/              # Author profiles (default.mdx = Tim Yu)
  style-guide.md        # Writing style extracted from existing posts — AI generators MUST follow this
  siteMetadata.js       # Site-wide config (title, URL, analytics, search, comments)
  projectsData.ts       # Projects page content
  headerNavLinks.ts     # Navigation links
  logo.svg              # Site logo
scripts/
  postbuild.mjs         # RSS generation — runs automatically after `next build` via the build script
  rss.mjs               # RSS feed generator — called by postbuild.mjs to create feed.xml
  generate_post.py      # (coming) AI blog post generator — reads EcoOrchestra artifacts, calls llm-router, writes .mdx
contentlayer.config.ts  # Content schema definition — Contentlayer2 reads this to type-check and process all .mdx files
layouts/                # Page layout components (PostLayout, ListLayout, AuthorLayout)
components/             # Reusable React components (Tag, SearchButton, etc.)
css/                    # Global styles (Tailwind base + custom)
public/
  static/images/        # Blog post images, organized by slug (e.g., public/static/images/<slug>/)
```

## Frontmatter schema

Defined in `contentlayer.config.ts`. Every `.mdx` file in `data/blog/` must include YAML frontmatter with these fields:

| Field | Type | Required | Default | Purpose |
|-------|------|----------|---------|---------|
| `title` | string | yes | — | Post title displayed on the page and in listings |
| `date` | date | yes | — | Publication date (ISO format, e.g., `2026-03-20`) |
| `lastmod` | date | no | — | Last modified date — shown when different from `date` |
| `tags` | list of strings | no | `[]` | Categorization tags — lowercase, hyphen-separated (e.g., `sitecore`, `xm-cloud`) |
| `draft` | boolean | no | `false` | When `true`, post is excluded from production builds |
| `summary` | string | no | — | Short description for listings and SEO meta tags |
| `images` | json | no | — | Array of image paths for Open Graph / structured data |
| `authors` | list of strings | no | — | Author slugs referencing files in `data/authors/` (default: `default` = Tim Yu) |
| `layout` | string | no | — | Override the default post layout component (e.g., `PostBanner`) |
| `bibliography` | string | no | — | Path to a `.bib` file for academic citations via rehype-citation |
| `canonicalUrl` | string | no | — | Canonical URL for cross-posted content — tells search engines where the original lives |
| `aiGenerated` | boolean | no | `false` | Flags AI-generated content — AI posts MUST set this to `true` for transparency |

### Computed fields (auto-generated by Contentlayer2, not set in frontmatter)

- `readingTime` — estimated reading time calculated from word count
- `slug` — URL slug derived from the file path
- `path` — full content path used for routing
- `filePath` — source file path for internal reference
- `toc` — table of contents extracted from headings
- `structuredData` — JSON-LD schema.org BlogPosting for SEO

## Commands

```bash
yarn dev        # Start dev server — uses cross-env to set INIT_CWD for Contentlayer2 path resolution
yarn build      # Production build + RSS generation — runs next build, then postbuild.mjs for feed.xml
yarn lint       # ESLint with auto-fix across all source directories (pages, app, components, lib, layouts, scripts)
yarn serve      # Serve the production build locally — useful for testing the built output before deploy
yarn analyze    # Bundle analysis — sets ANALYZE=true to generate @next/bundle-analyzer report
```

**Package manager**: yarn 3.6.1 (Plug'n'Play disabled — uses `.yarnrc.yml`). Always use `yarn` commands, never `npm`.

## Conventions

### Blog posts
- **Filename**: `<slug>.mdx` in `data/blog/` — the slug becomes the URL path (`/blog/<slug>`)
- **Images**: Place in `public/static/images/<slug>/` — reference as `/static/images/<slug>/filename.png`
- **Tags**: Lowercase, hyphen-separated (e.g., `sitecore`, `xm-cloud`, `azure-paas`)
- **Author**: Use `default` unless adding a guest author (references `data/authors/default.mdx` = Tim Yu)

### AI-generated posts
- MUST set `aiGenerated: true` in frontmatter — marks the post as AI-generated for transparency
- MUST set `draft: true` in frontmatter — AI posts always start as drafts for human review before publishing
- MUST follow the writing style guide at `data/style-guide.md` — ensures AI output matches Tim's voice and structure

### Code style
- Husky + lint-staged enforce formatting on commit — ESLint fixes JS/TS, Prettier formats all file types
- TypeScript throughout (`.ts`, `.tsx`)
- Tailwind CSS for styling — no custom CSS unless extending Tailwind utilities

## Ecosystem integration

ghostpen is part of the EcoOrchestra ecosystem. It consumes AI-generated content based on development activity across other projects.

### Content generation pipeline (coming)
1. **Source material**: EcoOrchestra artifacts (vision briefs in `decisions/vision-briefs/`, code reviews in `reviews/`, daily standups in `standups/`) provide raw material for blog topics
2. **LLM routing**: `scripts/generate_post.py` calls the llm-router HTTP API at `http://localhost:8321/route` to generate blog post content — llm-router handles provider selection and rate limiting
3. **Post-ship hook**: `~/.claude/hooks/generate-blog-post.sh` triggers `scripts/generate_post.py` after code is shipped in other ecosystem projects — automates the "write about what you built" workflow
4. **Output**: Generator writes `.mdx` files to `data/blog/` with `aiGenerated: true` and `draft: true` set automatically

### Dependencies
- **llm-router** (`C:/Repos/llm-router`): HTTP API for LLM calls — ghostpen's generator script is a consumer, not a direct dependency
- **EcoOrchestra** (`C:/Repos/EcoOrchestra`): Orchestration layer — provides the persona pipeline, planning, and artifact storage that feeds content generation
